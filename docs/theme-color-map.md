# Omarchy Color Map (Flat OneDark)

## Scope
This document maps where Omarchy reads colors from at runtime and how this repository generates those files.

## Theme Flow
1. Edit `theme.tokens.sh`.
2. Run `./scripts/build-theme.sh`.
3. Apply with `omarchy-theme-set flat-onedark` (or the installed theme name).
4. Omarchy copies the theme to `~/.config/omarchy/current/theme`, generates template files from `colors.toml`, and restarts affected components.

## Source Types
- Direct outputs from this repo: generated directly by `scripts/build-theme.sh`.
- Template outputs: generated by `omarchy-theme-set-templates` using `colors.toml` and Omarchy templates (`~/.local/share/omarchy/default/themed/*.tpl`).

## Waybar Policy
- Recommended default Omarchy mode:
  - Use `~/.config/waybar/config.jsonc` and `~/.config/waybar/style.css` from Omarchy defaults.
  - Keep theme color import from `~/.config/omarchy/current/theme/waybar.css`.
- Optional repo variant:
  - `waybar-theme/config.jsonc` and `waybar-theme/style.css` are kept as a custom layout/style pair.
  - These files are generated by `scripts/build-theme.sh`.

Reset to default Waybar mode:

```bash
omarchy-refresh-waybar
```

## Hardcoded Color Policy
- `theme.tokens.sh` is the only place to edit palette values.
- `scripts/build-theme.sh` must generate all repo files that encode colors.
- `waybar-theme/config.jsonc` calendar colors are tokenized via:
  - `WAYBAR_CALENDAR_MONTH_COLOR`
  - `WAYBAR_CALENDAR_WEEKDAY_COLOR`
  - `WAYBAR_CALENDAR_TODAY_COLOR`
