# File Ownership

## Why Ownership Matters

Most regressions in this repo come from the wrong layer writing the right file.

This document answers three questions for each surface:

- who writes it
- where it lives
- who must not write it

This repo should not add root-level copies of those files unless it is intentionally taking ownership away from Omarchy templates.

## Runtime Ownership Matrix

| Surface | Runtime path | Owner | Who must not write it |
| --- | --- | --- | --- |
| Active theme files | `~/.config/omarchy/current/theme/*` | Omarchy theme application | ad hoc local scripts |
| Repaired stock snapshot | `~/.config/waybar/themes/omarchy-default/*` | `./scripts/repair-waybar.sh` | manual overlay flows |
| Live Waybar files | `~/.config/waybar/config.jsonc`, `~/.config/waybar/style.css` | `./scripts/repair-waybar.sh` | manual Waybar edits |
| Starship runtime | `~/.config/starship.toml` | this theme apply flow | external tools |
| Omarchy source tree | `~/.local/share/omarchy/*` | Omarchy source repo | this theme |

## Allowed Mutation Zones For This Repo

This repo may write:

- `~/.config/omarchy/themes/flat-onedark`
- `~/.config/omarchy/current/theme`
- `~/.config/starship.toml`
- `~/.config/waybar/themes/omarchy-default`
- `~/.config/waybar/config.jsonc`
- `~/.config/waybar/style.css`

This repo may not write:

- anything under `~/.local/share/omarchy/`

## Maintenance Rule

If a surface is wrong, identify the owner first.

Use [runtime-surfaces.md](./runtime-surfaces.md) for the path map.
