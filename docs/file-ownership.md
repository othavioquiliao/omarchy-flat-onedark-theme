# File Ownership

## Why Ownership Matters

Most regressions in this repo come from the wrong layer writing the right file.

This document answers three questions for each surface:

- who writes it
- where it lives
- who must not write it

## Runtime Ownership Matrix

| Surface | Runtime path | Owner | Who must not write it |
| --- | --- | --- | --- |
| Active theme files | `~/.config/omarchy/current/theme/*` | Omarchy theme application | qbar and ad hoc local scripts |
| Repaired stock snapshot | `~/.config/waybar/themes/omarchy-default/*` | `./scripts/repair-waybar.sh` | qbar and manual overlay flows |
| Live Waybar files, stock-safe state | `~/.config/waybar/config.jsonc`, `~/.config/waybar/style.css` | `./scripts/repair-waybar.sh` | non-theme qbar or manual Waybar edits |
| Live Waybar files, qbar overlay enabled | `~/.config/waybar/config.jsonc`, `~/.config/waybar/style.css` | theme overlay scripts | non-theme qbar or manual Waybar edits |
| qbar overlay state | `~/.config/waybar/.flat-onedark-qbar-overlay.json` | this theme | qbar |
| qbar Waybar assets | `~/.config/waybar/qbar/*` | qbar | the repaired stock snapshot |
| qbar terminal helper | `~/.config/waybar/scripts/qbar-open-terminal` | qbar | the repaired stock snapshot |
| qbar settings and auth | `~/.config/qbar/*` | qbar | this theme |
| qbar cache | `~/.cache/qbar/*` | qbar | this theme |
| Starship runtime | `~/.config/starship.toml` | this theme apply flow | qbar |
| Omarchy source tree | `~/.local/share/omarchy/*` | Omarchy source repo | this theme and qbar |

## Allowed Mutation Zones For This Repo

This repo may write:

- `~/.config/omarchy/themes/flat-onedark`
- `~/.config/omarchy/current/theme`
- `~/.config/starship.toml`
- `~/.config/waybar/themes/omarchy-default`
- `~/.config/waybar/config.jsonc`
- `~/.config/waybar/style.css`
- `~/.config/waybar/.flat-onedark-qbar-overlay.json`

This repo may not write:

- anything under `~/.local/share/omarchy/`
- `~/.config/qbar/*`
- `~/.cache/qbar/*`

## qbar Boundary

The supported split is:

- theme-owned wiring
- qbar-owned runtime

This repo may own:

- whether the overlay is enabled
- how qbar modules are composed into live Waybar files
- when `qbar.css` is imported into the live style

This repo must not own:

- qbar settings
- qbar provider runtime
- qbar cache
- qbar business logic

## Maintenance Rule

If a surface is wrong, identify the owner first.

Use [runtime-surfaces.md](./runtime-surfaces.md) for the path map and [qbar-integration.md](./qbar-integration.md) for the overlay split.
